# AI Architect Practice Arena

[![ci](https://github.com/vpeetla-ai/ai-architect-practice-arena/actions/workflows/ci.yml/badge.svg)](https://github.com/vpeetla-ai/ai-architect-practice-arena/actions/workflows/ci.yml)
[![Org](https://img.shields.io/badge/GitHub-vpeetla--ai-blue)](https://github.com/vpeetla-ai)

A mock-interview practice layer for [ai-architect-interview-playbook](https://github.com/vpeetla-ai/ai-architect-interview-playbook):
write a real answer to one of its 35 questions, and get graded by **both OpenAI and Anthropic**
against that entry's own real Staff+/Principal rubric — using your own API key, which is never
stored on or sent to our servers.

## Why two judges, and why bring-your-own-key

Every playbook entry already encodes a specific, written rubric ("What's expected at each
level"); this tool reads that text directly rather than re-deriving it, so grading can never
drift from what the playbook itself says a good answer looks like. Both OpenAI and Anthropic
grade every attempt so their **agreement** becomes a real trust signal — a level only one judge
assigns is more informative than a single blended score. Bring-your-own-key means this org bears
zero direct API cost no matter how many people use the tool; see
[docs/adr/0001](docs/adr/0001-byok-judge-architecture.md) for the real security/architecture
trade-offs that decision required, including a real bug it caught (OpenAI's API silently blocks
direct browser calls; a minimal stateless proxy fixes it without ever storing a key).

## Architecture

```text
content/playbook/        — ai-architect-interview-playbook as a pinned git submodule (build-time only)
content/rubrics.json     — generated by scripts/build_rubrics.py, the single source of truth for grading
content/calibration/     — weak/strong reference answers per question, with expected verdict ranges
scripts/build_rubrics.py — parses the playbook's real rubric sections; fails loudly on any gap
backend/                 — FastAPI serving /questions and /questions/{id}/rubric only; never touches a key
frontend/lib/judge/      — OpenAIJudgeAdapter + AnthropicJudgeAdapter (mirrors aegisai's LLMGateway shape),
                            consensus merging, cost estimation (ported from agent-finops's pricing.py)
frontend/app/            — question picker, practice page, and a minimal stateless OpenAI proxy route
```

## Quick start

```bash
git clone --recurse-submodules https://github.com/vpeetla-ai/ai-architect-practice-arena.git
cd ai-architect-practice-arena

python3 scripts/build_rubrics.py                     # generates content/rubrics.json

cd backend && python3 -m venv .venv && .venv/bin/pip install -e ".[dev]"
.venv/bin/uvicorn practice_arena.main:app --port 8000 &

cd ../frontend && npm install
echo 'NEXT_PUBLIC_API_BASE=http://localhost:8000' > .env.local
npm run dev
```

Then open `http://localhost:3000`, pick a question, write an answer, paste your own OpenAI
and/or Anthropic API key, and grade it. Your key is used only to call the provider directly (or,
for OpenAI, through the minimal same-origin proxy documented in
[docs/adr/0001](docs/adr/0001-byok-judge-architecture.md)) — never stored, never logged.

To run the real calibration set against live providers yourself:

```bash
cd frontend
OPENAI_API_KEY=sk-... ANTHROPIC_API_KEY=sk-ant-... npm run calibrate
```
